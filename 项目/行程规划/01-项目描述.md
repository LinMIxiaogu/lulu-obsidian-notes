## 行程规划 AI Agent 系统（libai / 李白）

**项目描述**

面向 C 端用户的旅行规划 AI 助手，负责 AI 编排层的架构设计与核心功能开发。系统以对话形式理解用户旅行意图，自动规划多日行程，并整合酒店、交通、景点、天气等多维数据，通过 SSE 实时流式输出给前端。

**技术栈**

 LangGraph · PostgreSQL+Pgvector  · RAG , Redis · Elasticsearch ·  BERT，Langfuse

---

**关键技术亮点**

**1. 基于 LangGraph 的多 Agent 编排架构**

采用多层图结构，各 Agent 节点在运行时动态决定下一跳，支持并行扇出与汇聚。意图识别、业务执行、问题推荐等节点并发触发，行程规划阶段酒店、交通、天气并行查询后汇聚生成完整行程。

**2. 多级意图识别漏斗 + BERT 分类器微调（SFT）**

设计多级优先级漏斗，从规则路由、关键词匹配到模型分类层层过滤，高置信度结果直接短路，降低 LLM 调用频次。模型分类基于 RoBERTa-large 针对旅行场景做监督微调。

**3. RAG + ES 双路召回的路线知识库**

从海量小红书帖子中，经 LLM 分类、路线结构提取、质量审核的多级漏斗沉淀出高质量路线知识库。召回时语义侧（向量检索）与关键词侧（ES BM25）双路并行，结果融合，两路互补提升召回率与排序质量。

**4. STD / NANO 双档上下文压缩**

多轮对话中，按节点职责分级管理上下文：业务执行节点使用完整上下文保证生成质量；意图识别与思考节点将历史 AI 回复压缩为单句摘要，用户消息原文保留。分级压缩在保证意图识别精度的同时大幅降低 token 消耗。

**5. LLM-as-Judge 双轨评测体系**

构建两套评测流程：在线侧对生产流量自动采样，由 LLM 从路线合理性、地理顺序、意图满足度等多维度打分，持续监控行程质量；离线侧基于人工标注数据集做功能回归，对比实际输出与标准答案，输出通过率，驱动模型与 prompt 迭代。
