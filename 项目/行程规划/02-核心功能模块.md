## 模块 1：路线规划（Route Planning）

### 实现路径

```
用户输入旅游需求
        │
travel_intent_sector_node       ← 提取结构化意图
        │
    TravelIntention 对象
    {departure: "上海", arrival: "三亚", days: 5, adult: 2, ...}
        │
travel_route_planner_node       ← 调用路线生成引擎
        │
    ┌───┴───────────────────────────────────────┐
    │ 方式1: 召回路线       │ 方式2: AI 生成路线  │
    │ 从历史数据库检索       │ 调用 LLM 生成路线  │
    │ 相似行程             │ 提示词: generate_  │
    │                     │ travel_guide.md   │
    └──────────────────────┴────────────────────┘
        │
    DayRouteResult 列表（每天的路线）
        │
travel_planner_node             ← 调用 route_store 落库
        │
    route_id → 返回给前端
```

### 核心数据结构

```python
class TravelIntention(BaseModel):
    departure: str        # 出发地，如 "上海"
    arrival: str          # 目的地，如 "三亚"
    start_date: str       # 开始日期 "2026-04-01"
    end_date: str         # 结束日期 "2026-04-05"
    days: int             # 行程天数
    adult_count: int      # 成人数量
    child_count: int      # 儿童数量
    select_tags: List[str]          # 兴趣标签: ["海滩", "美食"]
    trip_style_dict: Optional[dict] # 风格: {"亲子": true}
    select_pois: List[str]          # 用户指定必去景点 ID

class DayRouteResult(BaseModel):
    day: int              # 第几天
    theme: str            # 当天主题，如 "海边休闲日"
    route_text: str       # 路线描述："三亚湾→亚龙湾→大东海"
    highlights: str       # 亮点推荐
    full_desc: str        # 完整描述
    spots: List[PoiSpot]  # 景点详情列表
```

### 生成的路线格式（LLM 输出）

```
Day 1 | 主题：海滩初体验
路线：三亚湾海滩 → 大东海 → 椰梦长廊
亮点：三亚湾是市区最近的海滩，晚上可以看日落
景点详情：
  - 三亚湾海滩（上午 9:00-11:00）免费开放，建议早去避开人流
  - 大东海（下午 14:00-17:00）冲浪胜地，可租冲浪板
...
```

## 模块 2：酒店服务（Hotel Service）

### 两种场景

| 场景         | 节点                          | 触发时机          |
| ---------- | --------------------------- | ------------- |
| 行程规划中的酒店推荐 | `travel_hotel_planner_node` | 用户请求完整行程规划    |
| 独立酒店咨询     | `hotel_consult_sector_node` | 用户单独问"有什么好酒店" |

### 酒店搜索流程

```
用户文本："三亚亚龙湾附近有什么亲子酒店，500元以内的"
        │
    NLP 条件提取 (text_to_hotel_list_filters.md)
        │
    {
      city: "三亚",
      area: "亚龙湾",
      type: "亲子",
      max_price: 500,
      check_in: "2026-04-01",
      check_out: "2026-04-03"
    }
        │
    调用携程酒店搜索 API (ctrip_service.py)
        │
    返回酒店列表 → 格式化 → 发送 HOTEL_CARD 事件
```

### 支持的过滤维度

- 位置/商圈（如亚龙湾、三亚湾）
- 价格区间（min/max）
- 星级评分（3星/4星/5星）
- 酒店类型（亲子、度假、商务）
- 设施（游泳池、儿童乐园、海景）

## 模块 3：交通服务（Transport Service）

### 交通查询流程

```
用户："明天从上海到三亚有没有直飞，价格多少"
        │
transport_consult_sector_node
        │
    ┌───┴──────────┐
    │              │
机票搜索        火车票搜索
(flight_search) (train_search)
    │              │
    └──────────────┘
        │ 并行查询
        ▼
    结果合并 → TRANSPORT_CONSULT_SECTOR_BY_RECOMMEND 事件
```

### 特色功能

| 功能 | 说明 |
|------|------|
| **低价日历** | 展示一个月内每天最低票价 |
| **价格趋势** | 预测价格走势（涨/降）|
| **Bug 价格** | 自动识别异常低价机票 |
| **直飞偏好** | 支持"只要直飞"的条件过滤 |
| **舱位选择** | 经济舱/商务舱/头等舱 |

## 模块 4：小红书笔记导入（XHS Import）

这是一个**特色功能**，允许用户把小红书上看到的旅游攻略直接导入为可执行的行程。

### 实现流程

```
用户发送笔记链接或笔记内容
        │
travel_xhs_import_sector_node   ← 识别要导入的笔记
        │
    crawler.py 爬取笔记内容
    + Readability 提取正文
        │
    LLM 解析笔记 → 提取结构化路线
        │
    ┌──────────────────────────────────────────┐
    │  并行处理（与主流程相同）                   │
    ├─ xhs_import_route_planner_node           │
    ├─ xhs_import_transport_planner_node       │
    ├─ xhs_import_hotel_planner_node           │
    └─ xhs_import_weather_planner_node         │
    └──────────────────────────────────────────┘
        │
travel_xhs_import_planner_node  ← 汇总落库
        │
    返回 route_id
```

### 技术要点

- 使用 `Readability` 库过滤广告、导航等无关内容，提取正文
- Jina API 辅助提取内容
- LLM 将非结构化的笔记文字转化为标准的 `TravelIntention + DayRouteResult` 结构

## 模块 5：路线调整（Adjust Route）

当用户对已生成的行程不满意时，可以通过自然语言调整。

### 示例交互

```
用户：把第二天的大东海换成蜈支洲岛
系统：[读取当前 route_id 对应的路线] → [LLM 理解修改意图]
     → [局部修改 Day 2 景点] → [保存新版本路线] → [返回修改确认]
```

### 实现关键

- 请求携带 `route_id` 参数，表示要修改哪个行程
- `adjust_route_sector_node` 读取 RouteState 中的当前路线
- LLM 接收"当前路线 + 修改指令"，输出修改后的路线
- 再次调用 `route_store` 更新路线

## 模块 6：打卡机位生成（Guide Shot）

独立 API 功能，为景点或小红书笔记推荐最佳拍照位置。

**调用入口：** `POST /api/guide_shot`

```python
# 请求
{
    "taskId": "task_xxx",
    "noteId": "xhs_note_id"  # 小红书笔记 ID
}

# 响应
{
    "success": true,
    "data": {
        "shots": [
            {"location": "亚龙湾山顶", "angle": "朝东南方向45度俯拍", ...}
        ]
    }
}
```

## 模块 7：问题推荐（Question Recommend）

在规划完成后，系统会根据对话上下文智能推荐用户可能会继续问的问题。

**使用提示词：** `prompts/question_recommender.md`

**输出形式（QUESTION_RECOMMEND 事件）：**

```json
{
    "event": "QUESTION_RECOMMEND",
    "data": [
        "需要帮您安排三亚的接送机吗？",
        "要推荐亚龙湾附近的海鲜餐厅吗？",
        "需要查看这个时间段的天气预报吗？"
    ]
}
```

**生成逻辑：** 分析用户刚刚规划的内容 + 对话历史 + 常见后续需求 → 选择 3-5 个最相关的问题
