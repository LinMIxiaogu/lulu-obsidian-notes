## 一、 数据结构与语法基础

### 1. Python 中 `is` 和 `==` 的区别？

- `==`：比较的是两个对象的内容（Value）是否相等。
    
- `is`：比较的是两个对象的内存地址（ID）是否相同，即是否是同一个对象。
    

### 2. 列表（List）和元组（Tuple）的区别？

- **不可变性**：List 是可变的（mutable），Tuple 是不可变的（immutable）。
    
- **性能**：Tuple 由于不可变，内存开销更小，遍历速度略快。
    
- **用途**：List 用于存储同质数据的集合；Tuple 常用于结构化数据（如数据库记录）或作为字典的 Key（List 不能作为 Key）。
    

### 3. Python 字典（Dict）的底层实现原理？

- 底层是 **哈希表（Hash Table）**。
    
- 通过哈希函数将 Key 映射到桶（bucket），查找、插入、删除的平均时间复杂度均为 $O(1)$。
    
- Python 3.7+ 后的 Dict 是有序的，通过维护一个额外的索引数组来实现。
    

### 4. 什么是可变对象和不可变对象？

- **不可变**：`int`, `float`, `string`, `tuple`, `frozenset`。修改时会创建新对象。
    
- **可变**：`list`, `dict`, `set`。可以原地修改。
    

### 5. 深拷贝（Deep Copy）和浅拷贝（Shallow Copy）的区别？

- **浅拷贝**：只拷贝父对象，不会拷贝对象内部的子对象（子对象依然是指向同一个引用）。
    
- **深拷贝**：完全递归拷贝，原始对象与新对象完全独立，互不影响。

## 二、 高级特性与函数式编程

### 6. 什么是装饰器（Decorator）？其作用是什么？

- 装饰器本质上是一个闭包函数，它允许在不修改原函数代码的基础上，给函数增加新功能。
    
- 常见应用：日志记录、权限校验、性能测试、缓存（memoization）。
    

### 7. 解释 Python 的闭包（Closure）

- 在一个外部函数内定义了一个内部函数，内部函数引用了外部函数的变量，并且外部函数返回了这个内部函数。
    
- 即使外部函数执行结束，内部函数依然能访问那些变量。
    

### 8. `*args` 和 `**kwargs` 的含义？

- `*args`：用于接收任意数量的**位置参数**，在函数内部表现为 Tuple。
    
- `**kwargs`：用于接收任意数量的**关键字参数**，在函数内部表现为 Dict。
    

### 9. 迭代器（Iterator）和生成器（Generator）的区别？

- **迭代器**：实现了 `__iter__` 和 `__next__` 方法的对象。
    
- **生成器**：一种特殊的迭代器，使用 `yield` 关键字定义。它不会一次性生成所有数据，而是“惰性计算”，极大地节省了内存。
    

### 10. 什么是匿名函数（Lambda）？

- 单行的、没有名字的函数。常配合 `map()`, `filter()`, `sorted()` 等高阶函数使用。

## 三、 面向对象编程（OOP）

### 11. Python 中的 `self` 和 `__init__` 是什么？

- `self`：指向实例对象本身，通过它访问类的方法和属性。
    
- `__init__`：构造方法，在创建对象实例时自动调用，用于初始化属性。
    

### 12. `__new__` 和 `__init__` 的区别？

- `__new__`：真正创建实例的方法，返回一个实例对象，是静态方法。
    
- `__init__`：初始化实例的方法，不返回值。
    
- 先执行 `__new__` 拿到对象，再由 `__init__` 进行修饰。
    

### 13. Python 如何实现多重继承？什么是 MRO？

- Python 支持一个类继承多个父类。
    
- **MRO (Method Resolution Order)** 是方法解析顺序。Python 3 使用 **C3 算法** 来确定继承链中方法的调用顺序，可以通过 `ClassName.mro()` 查看。
    

### 14. 什么是元类（Metaclass）？

- 元类是“类的类”。类定义了对象的行为，而元类定义了类的行为。`type` 是 Python 中所有类的默认元类。
    

### 15. 鸭子类型（Duck Typing）是什么？

- “如果它走起路来像鸭子，叫起来也像鸭子，那它就是鸭子”。
    
- Python 不关心对象的具体类型，只关心它是否具有特定的方法或属性。

## 四、 底层原理与并发

### 16. 什么是 GIL（全局解释器锁）？

- GIL 保证在同一时刻只有一个线程在执行 Python 字节码。
    
- **后果**：即使在多核 CPU 上，Python 的多线程也无法实现真正的并行。
    
- **对策**：CPU 密集型任务改用**多进程（multiprocessing）**；IO 密集型任务使用多线程或 **协程（asyncio）**。
    

### 17. Python 的垃圾回收机制（GC）？

- **引用计数**（主要）：每个对象维护一个引用计数，为 0 时立即回收。
    
- **标记-清除**：解决循环引用问题。
    
- **分代回收**：基于“对象存在越久，越不容易被回收”的假设，将对象分为三代提高效率。
    

### 18. Python 是如何运行的？（解释型 vs 编译型）

- Python 源码 $\rightarrow$ 字节码（.pyc） $\rightarrow$ Python 虚拟机（PVM）逐行解释执行。
    

### 19. 谈谈多线程、多进程和协程？

- **多进程**：利用多核并行，内存隔离，开销大。
    
- **多线程**：并发执行，共享内存，受 GIL 限制。
    
- **协程**：用户态的轻量级线程（单线程内切换任务），通过 `async/await` 实现，极其高效地处理高并发 IO。
    

### 20. 什么是魔法方法（Magic Methods）？

- 以双下划线开头和结尾的方法（如 `__str__`, `__len__`, `__call__`）。
    
- 它们允许对象实现内置行为，如 `a + b` 实际上是调用了 `a.__add__(b)`。
