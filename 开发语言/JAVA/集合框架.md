# Java 集合框架面试题

## Q1. Java 集合体系概览

**两大接口：** `Collection` 和 `Map`

**四大类（List/Set/Queue 都继承 Collection 接口）：**

| 接口 | 特性 | 常见实现类 |
|------|------|-----------|
| List | 有序、可重复 | ArrayList、LinkedList |
| Set | 无序、不可重复 | HashSet、TreeSet |
| Queue | 先进先出队列 | ArrayDeque |
| Map | 键值对映射 | HashMap、TreeMap |

以上都是**线程不安全**的集合。

**线程安全的集合：**
- `Vector`、`Hashtable`：古老 API，性能低，已基本不再使用
- `Collections.synchronizedXxx()`：包装方法将线程不安全集合转为线程安全集合
- 并发包（JUC）高效并发容器（推荐）：
  - `ConcurrentHashMap`：降低锁粒度
  - `CopyOnWriteArrayList`：写时复制技术
  - Lock 接口实现的阻塞队列

---

## Q2. HashMap 底层原理

**数据结构：** 数组 + 链表 + 红黑树

**工作原理：** 通过哈希算法得到哈希值确定插入位置，如果位置碰撞，则加入对应槽的链表中

**扩容机制（三种情况触发）：**
1. 插入数组时发现数组为空，扩容
2. 插入链表后，链表长度 > 8 时：
   - 数组容量 < 64：扩容
   - 数组容量 ≥ 64：链表转换为红黑树
3. 插入后，元素比例超过阈值：`数据/数组 > 0.75`，扩容

**扩容方式：** 通过 2^n 扩容，位运算方便扩容计算

## Q3. HashMap 和 Hashtable 的区别

| 特性 | HashMap | Hashtable |
|------|---------|-----------|
| 线程安全 | 不安全，性能好 | 安全，性能低 |
| null 键/值 | key 和 value 都可以为 null | 都不可以为 null |
| 推荐度 | 推荐使用 | 不推荐（已过时） |

**多线程环境下推荐使用 `ConcurrentHashMap`**

---

## Q4. HashMap 线程不安全问题及解决

**问题：** 多线程 put 时可能造成数据覆盖

**解决方案：**
1. `Collections.synchronizedMap()` 包装为线程安全集合
2. 使用 `ConcurrentHashMap`（更推荐，降低锁粒度，提升并发能力）

---

## Q5. ConcurrentHashMap

**数据结构：** 分段数组 + 链表 + 红黑树

**线程安全实现（五点）：**
1. 初始化数组和头结点时**不加锁**，通过 CAS 方式进行原子替换
2. **插入数据**时，锁住头节点
3. **查询数据**时，不加锁，性能好
4. **扩容**时加锁，支持多个线程对数组同时扩容，提高并发能力
5. 扩容时，仍然可以查找数据

## Q6. ArrayList 原理

- 底层是一个 `Object` 数组
- 初始化时是一个空数组，第一次添加元素时设置数组大小为 **10**
- 之后每次扩容以 **1.5 倍**扩容
- `Set`、`List`、`Queue` 都是 Collection 接口的子接口，都继承了 `iterator()` 方法，具备迭代能力

## Q7. ArrayList 和 LinkedList 的区别

| 特性 | ArrayList | LinkedList |
|------|-----------|-----------|
| 底层数据结构 | 数组 | 双向链表 |
| 随机访问（查找）| 快 O(1) | 慢 O(n) |
| 增删效率 | 低 O(n) | 高 O(1) |
| 内存空间 | 少 | 多（存储前后指针）|

## Q8. List 和 Set 的区别

| 特性 | List | Set |
|------|------|-----|
| 顺序 | 有序 | 无序 |
| 重复 | 可重复 | 不可重复 |

- `TreeSet` 支持有序（自然排序和定制排序），底层依靠 `TreeMap` 实现

## Q9. 线程安全集合汇总

**推荐程度（从高到低）：**

1. **ConcurrentHashMap**：降低锁粒度，高并发性能（最推荐）
2. **CopyOnWriteArrayList**：写时复制，读操作不加锁
3. **Collections.synchronizedXxx()**：包装方法，简单但性能一般
4. **Vector / Hashtable**：古老 API，性能低，不推荐

## Q10. HashMap put() 和 get() 流程

**put() 流程：**
1. 通过扰动哈希函数计算 key 应该放在哪个桶（`index = (n-1) & hash`）
2. 如果桶上没有元素，直接插入
3. 如果发生哈希冲突，判断是红黑树还是链表：
   - 红黑树：使用红黑树插入方法
   - 链表：遍历，如果找到相同 key 则替换 value；否则尾插
4. 链表长度 > 8 且数组容量 >= 64，链表转为红黑树
5. 插入后判断哈希表中桶的占用是否达到阈值（容量 × 0.75），达到则扩容

**get() 流程：**
1. 通过 key 的哈希算法找到映射的桶
2. 桶上首个元素是要找的 key，直接命中返回
3. 否则判断是红黑树还是链表，按各自查找方法查找 key

## Q11. HashMap 扰动函数

**作用：** 降低哈希碰撞的概率

**原理：**
```java
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}
```
1. 先拿到 key 的 `hashCode()`，是一个 32 位的 int 值
2. 让高 16 位与低 16 位进行**异或**操作
3. 增加低位的随机性，降低哈希碰撞的概率

**为什么需要扰动：** HashMap 数组长度通常不大，`(n-1) & hash` 实际上只用到了哈希值的低位。扰动函数让高位信息也参与到槽位计算中，减少碰撞。
