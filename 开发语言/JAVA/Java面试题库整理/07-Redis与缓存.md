# Redis 与缓存

返回索引：[[00-Java面试题库索引]]

## Redis 是什么

- 基于键值对的 NoSQL 数据库。
- 数据主要在内存中，读写快。
- 常用于缓存、排行榜、分布式锁、会话、计数器等场景。

## 常见数据类型

- `String`
- `Hash`
- `List`
- `Set`
- `ZSet`

## Redis 为什么快

- 基于内存
- 单线程执行核心命令，减少锁竞争
- IO 多路复用

## 单线程架构

- 核心读写命令由单线程处理。
- 持久化、异步删除等操作可由其他线程辅助。

## 持久化策略

### RDB

- 周期性快照。
- 文件小，恢复快。
- 可能丢失最后一次快照后的数据。

### AOF

- 追加写命令日志。
- 数据更安全。
- 文件更大，恢复更慢。

## 主从同步

### 全量同步

- 首次同步时，主节点生成 RDB 发给从节点。
- 同步快照期间的写命令再补发。

### 增量同步

- 后续只同步偏移量之后的增量命令。

## 过期删除与淘汰策略

- 惰性删除
- 定期删除
- 内存淘汰策略，如 LRU、LFU 等

## 缓存穿透、击穿、雪崩

### 缓存穿透

- 查询不存在的数据，请求直接打到数据库。
- 常见解法：布隆过滤器、缓存空值、参数校验。

### 缓存击穿

- 热点 key 失效瞬间，大量请求打穿数据库。
- 常见解法：互斥锁、热点不过期、逻辑过期。

### 缓存雪崩

- 大量 key 同时过期或 Redis 故障。
- 常见解法：过期时间打散、集群、限流降级。

## 缓存与数据库双写一致性

- 常见思路：
- 先更缓存再更库
- 先更库再更缓存
- 先删缓存再更库
- 更常用：先更新数据库，再删除缓存
- 可配合重试、延迟双删等手段降低不一致窗口

## Redis 高可用

- 主从复制
- 哨兵模式
- Redis Cluster

## Redis 分布式锁

- 基础思路：`SET key value NX EX`
- value 一般是唯一随机值，释放锁前要校验值是否匹配
- 删除锁的动作通常借助 Lua 脚本保证原子性
