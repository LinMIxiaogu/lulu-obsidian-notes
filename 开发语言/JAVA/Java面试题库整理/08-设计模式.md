# 设计模式

返回索引：[[00-Java面试题库索引]]

## 常见设计模式

- 单例模式
- 工厂模式
- 代理模式
- 适配器模式
- 模板方法模式
- 装饰器模式
- 策略模式
- 状态模式

## 单例模式

### 定义

- 一个类在系统中只存在一个实例。

### 优点

- 节省内存
- 避免重复创建

### 常见写法

- 饿汉式：类加载时创建实例。
- 懒汉式：首次使用时创建实例。
- 双重检查锁：兼顾懒加载和线程安全。
- 静态内部类：推荐度较高，简洁且线程安全。

## 工厂模式

### 目标

- 把对象创建过程封装起来，调用者不用关心创建细节。

### 简单工厂

- 工厂方法根据参数返回不同对象。
- 问题是新增产品常需要修改工厂本身。

### 工厂方法

- 每种产品对应一个工厂。
- 更符合开闭原则。

### 抽象工厂

- 可以创建一族相关产品，而不只是单一对象。

## 代理模式

- 通过代理对象控制对目标对象的访问。
- 常见用途：权限校验、日志、事务、远程调用。
- Spring AOP 底层常用代理思想。

## 适配器模式

- 让原本接口不兼容的类能一起工作。
- 本质是做接口转换。

## 模板方法模式

- 在父类定义流程骨架。
- 子类只实现或覆盖某些步骤。
- 用于复用通用流程，控制扩展点。
