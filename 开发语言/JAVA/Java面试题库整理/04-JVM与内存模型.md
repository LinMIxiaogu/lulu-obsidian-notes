# JVM 与内存模型

## JVM 整体认识

- JVM 是 Java 跨平台的关键。
- 主要组成：
- 类加载子系统
- 运行时数据区
- 执行引擎

## 运行时数据区

- 程序计数器
- 虚拟机栈
- 本地方法栈
- 堆
- 方法区

### 线程私有

- 程序计数器
- 虚拟机栈
- 本地方法栈

### 线程共享

- 堆
- 方法区

## JMM

- JMM 是 Java 内存模型，不等于 JVM 内存区域划分。
- 它定义了线程与主内存、工作内存之间的抽象关系。
- 目标是屏蔽底层硬件差异，解决可见性、原子性、有序性问题。

## 类加载机制

### 流程

- 加载
- 验证
- 准备
- 解析
- 初始化
- 使用
- 卸载

### 双亲委派模型

- 类加载请求先委派给父加载器。
- 父加载器无法完成时，子加载器再尝试。
- 好处是避免核心类被重复加载或篡改。

## 类的实例化过程

- 类加载
- 分配内存
- 零值初始化
- 设置对象头等信息
- 执行构造方法

## 垃圾回收机制

- 关注三个问题：
- 哪些对象是垃圾
- 何时回收
- 怎样回收

## 如何判断对象是否可回收

### 可达性分析

- 从 GC Roots 出发沿引用链搜索。
- 不可达对象会被判定为可回收。
- 常见 GC Roots：
- 栈帧中的引用
- 静态变量引用
- 常量引用
- 锁持有对象

## 垃圾回收算法

- 标记清除
- 标记复制
- 标记整理

### 常见使用场景

- 新生代：多用标记复制
- 老年代：多用标记清除或标记整理

## 四种引用

- 强引用：只要存在，一般不会被回收。
- 软引用：内存紧张时回收。
- 弱引用：GC 时基本都会回收。
- 虚引用：主要用于跟踪对象被回收的过程。

## CMS

- 面向低停顿的老年代收集器。
- 基于标记清除。
- 主要流程：
- 初始标记
- 并发标记
- 重新标记
- 并发清除
- 缺点是会产生内存碎片。

## G1

- 将堆划分为多个 Region。
- 根据收益和停顿目标选择回收区域。
- 整体上兼顾吞吐和停顿。
- 局部有复制，整体有整理。

## 内存溢出和内存泄漏

### 内存溢出 OOM

- 申请的内存超过系统可提供的范围。
- 常见原因：大对象过多、死循环造对象、堆设置过小。

### 内存泄漏

- 无用对象仍被引用，导致无法回收。
- 常见于容器误用、资源未关闭、ThreadLocal 未清理。
