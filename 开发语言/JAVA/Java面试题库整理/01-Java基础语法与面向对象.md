# Java 基础语法与面向对象

返回索引：[[00-Java面试题库索引]]

## Java 的特点和优点

- 面向对象：封装、继承、多态。
- 平台无关：一次编译，到处运行，依赖 JVM 执行字节码。
- 自动垃圾回收，减少手动内存管理成本。
- 相比 C++，没有多继承和显式指针复杂度。
- 类库丰富，支持网络编程、多线程，生态成熟。

## 基本数据类型与引用类型

- 八大基本类型：`byte`、`short`、`int`、`long`、`float`、`double`、`char`、`boolean`。
- 引用类型：数组、类、接口、`String` 等。
- 基本类型值通常直接存储，引用类型变量保存的是对象地址。
- 装箱：基本类型转包装类。
- 拆箱：包装类转基本类型。

## 抽象类和接口的区别

- 相同点：都不能直接实例化，都可包含抽象方法。
- 抽象类用 `abstract class`，接口用 `interface`。
- 类只能单继承抽象类，但可实现多个接口。
- 抽象类可有成员变量、构造器、普通方法。
- 接口更偏规范，抽象类更偏模板。

## final 与 static

### final

- 修饰类：不可继承。
- 修饰方法：不可重写。
- 修饰变量：只能赋值一次。
- 修饰引用：地址不可变，但对象内容可变。

### static

- 修饰成员变量、方法、代码块、内部类。
- 静态成员属于类，类加载时创建。
- 静态方法里没有 `this`，不能直接访问实例成员。

## String 相关

### String 的特点

- `String` 是 `final` 类，不可继承。
- 字面量会进入字符串常量池。
- `new String("x")` 可能同时涉及常量池和堆对象。

### 常用方法

- 大小写转换：`toUpperCase()`、`toLowerCase()`
- 截取：`substring()`
- 查找：`indexOf()`
- 去空白：`trim()`
- 替换：`replaceFirst()`、`replaceAll()`

### String、StringBuffer、StringBuilder

- `String`：不可变，修改会产生新对象。
- `StringBuffer`：可变，线程安全。
- `StringBuilder`：可变，线程不安全但性能更好。
- 常见操作：`append()`、`insert()`、`reverse()`

## ==、equals()、hashCode()

- `==` 比较基本类型的值，比较引用类型的地址。
- `equals()` 默认也是地址比较，常被重写为内容比较。
- `hashCode()` 用于散列定位。
- 约束：`equals()` 相等的两个对象，`hashCode()` 必须相等。
- 所以重写 `equals()` 时一般必须同时重写 `hashCode()`。

## Java 8 新特性

- Lambda 表达式
- Stream API
- 新日期时间 API
- 接口默认方法
- 方法引用

## 自动装箱与自动拆箱

- 自动装箱：基本类型直接赋值给包装类。
- 自动拆箱：包装类直接赋值给基本类型。
- 本质是编译器帮你插入转换逻辑。

## 异常处理机制

- `try`：包裹业务代码。
- `catch`：捕获并处理异常。
- `finally`：通常用于资源释放。
- `throws`：写在方法签名上，声明可能抛出的异常。
- `throw`：在方法体中主动抛出异常对象。
- 未被处理的异常会一直向上抛到 JVM，最终打印堆栈并终止程序。

## 面向对象三大特征

### 封装

- 隐藏内部状态，只暴露必要操作。
- 有助于控制访问、提高可维护性。

### 继承

- 提高复用性。
- 也会带来更强耦合。

### 多态

- 父类引用指向子类对象。
- 调用重写方法时表现为运行时多态。

## 重载和重写

### 重载

- 同一类中方法名相同，参数列表不同。

### 重写

- 子类对父类方法的重新实现。
- 返回值范围不能放大。
- 抛出异常范围不能放大。
- 访问权限不能缩小。
- `final`、`private` 方法和构造器不能被重写。

## 访问修饰符

- 权限顺序：`private < default < protected < public`
- `private`：仅当前类。
- `default`：同包可见。
- `protected`：同包或子类可见。
- `public`：任何地方可见。

## 泛型与泛型擦除

- 泛型用于在编译期做类型约束。
- 能减少强制类型转换和运行时类型错误。
- Java 泛型本质是伪泛型，编译后会发生类型擦除。

## 反射

- 可在运行时获取类信息、字段、方法、构造器。
- 可动态创建对象和调用方法。
- 动态代理底层常依赖反射。
- 优点是灵活，缺点是破坏封装、性能较低、存在安全风险。
