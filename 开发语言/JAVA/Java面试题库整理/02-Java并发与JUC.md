# Java 并发与 JUC

返回索引：[[00-Java面试题库索引]]

## 线程与进程

- 进程是资源分配的基本单位。
- 线程是 CPU 调度的基本单位。
- 一个进程可包含多个线程。
- 线程共享进程内的堆和方法区，但程序计数器、栈等私有。

## 多线程的价值与风险

- 优点：提高 CPU 利用率、提升响应速度、线程切换成本低于进程。
- 风险：死锁、竞态条件、上下文切换开销、调试复杂。

## 线程创建方式

- 继承 `Thread`
- 实现 `Runnable`
- 实现 `Callable` 配合 `FutureTask`
- 使用线程池

## 线程状态

- 新建
- 就绪 / 运行
- 阻塞
- 等待
- 超时等待
- 终止

## wait() 和 sleep() 区别

- `wait()` 属于 `Object`，会释放锁。
- `sleep()` 属于 `Thread`，不会释放锁。
- `wait()` 需要 `notify()` / `notifyAll()` 唤醒。
- `sleep()` 到时间自动恢复就绪。

## 线程安全三要素

- 原子性：操作不可分割。
- 可见性：一个线程修改后，其他线程能及时看到。
- 有序性：程序执行顺序不会因重排序破坏语义。

## 如何保证线程安全

- `synchronized`：保证原子性、可见性、有序性。
- `volatile`：保证可见性和一定的有序性。
- `Lock`、原子类：更灵活地保证并发安全。

## synchronized

### 用法

- 修饰实例方法：锁当前对象。
- 修饰静态方法：锁 `Class` 对象。
- 修饰代码块：锁指定对象。

### 原理

- 基于对象头中的监视器信息。
- 锁会经历无锁、偏向锁、轻量级锁、重量级锁等升级过程。

## Lock 与 synchronized 区别

- `synchronized` 自动释放锁。
- `Lock` 需要手动释放。
- `Lock` 支持可中断、超时、尝试获取锁。
- `Lock` 能配合多个 `Condition` 做更灵活的线程通信。

## 线程同步与通信

### 线程同步方式

- `synchronized`
- `Lock`

### 线程通信方式

- `Monitor`：`wait()`、`notify()`、`notifyAll()`
- `Condition`：`await()`、`signal()`、`signalAll()`

## 常见锁及原理

- `synchronized`：基于对象头和监视器。
- `Lock`：通常基于 AQS 实现。
- 悲观锁：先加锁再操作。
- 乐观锁：操作时不锁，提交时校验冲突，典型是 CAS。

## AQS

- AQS 是构建锁和同步器的基础框架。
- 内部维护一个状态值 `state`。
- 内部维护 FIFO 同步队列。
- 很多 `Lock`、同步工具类都基于 AQS。

## volatile

- 保证变量的可见性。
- 通过内存屏障禁止部分指令重排。
- 不保证复合操作的原子性。

## ThreadLocal

- 为每个线程维护独立变量副本。
- 常用于保存用户上下文、事务上下文、格式化工具等。
- 在线程池环境下必须注意 `remove()`，否则可能内存泄漏。

## JUC

- 即 `java.util.concurrent`
- 重点包括：
- 原子类
- 锁
- 线程池
- 并发容器
- 同步工具

## CAS

- Compare And Swap，比较并交换。
- 常见使用点：
- 原子类
- AQS
- 并发容器

## 线程池

### 核心参数

- 核心线程数
- 最大线程数
- 工作队列
- 空闲线程存活时间
- 拒绝策略

### 优点

- 线程复用
- 降低创建销毁成本
- 提高任务响应效率

### 注意点

- 不建议直接使用 `Executors` 默认工厂创建业务线程池。
- 原因是其无界队列等默认配置可能引发 OOM。
