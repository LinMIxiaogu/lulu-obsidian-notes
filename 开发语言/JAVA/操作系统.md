# 操作系统面试题

## Q1. 进程调度算法

| 算法 | 说明 | 特点 |
|------|------|------|
| **先来先服务（FCFS）**| 按进程到达顺序执行 | 简单，但长进程会阻塞短进程 |
| **最短作业优先（SJF）**| 优先执行执行时间最短的进程 | 吞吐量高，但可能导致长进程饥饿 |
| **高响应比优先（HRRN）**| 响应比 = (等待时间 + 服务时间) / 服务时间，取最高的 | 兼顾短进程和等待时间长的进程 |
| **时间片轮转（RR）**| 每个进程分配固定时间片，轮流执行 | 公平，适合交互式系统 |
| **最高优先级（Priority）**| 优先执行优先级最高的进程 | 可能导致低优先级进程饥饿 |
| **多级反馈队列**| 多个优先级队列，队列内先来先服务，优先级越高时间片越短，未完成的进程降级 | 自适应，兼顾各种类型进程 |

## Q2. 进程、线程、协程的区别

| 特性 | 进程 | 线程 | 协程 |
|------|------|------|------|
| 定义 | 资源分配的基本单位 | CPU 任务调度的基本单位 | 用户态的轻量级线程 |
| 资源 | 独享完整的资源（代码、数据、堆、文件描述符等）| 共享进程资源，独有寄存器和栈 | 共享线程资源 |
| 并发 | 并发性低 | 并发性高 | 并发（一个线程内串行）|
| 并行 | 可以真正并行 | 多核可以并行 | 一个线程内多个协程无法并行 |
| 调度 | 操作系统调度，抢占式 | 操作系统调度，抢占式 | 用户程序控制，非抢占式（需主动让出）|
| 上下文切换 | 开销大（需切换虚拟内存等）| 开销中等（内核级切换）| 开销极小（用户态切换，无内核切换）|
| 创建开销 | 大 | 中等 | 小 |
| 崩溃影响 | 不影响其他进程 | 会导致进程崩溃 | 影响同一线程内其他协程 |

## Q3. 进程通信方式（IPC）

| 方式 | 说明 | 特点 |
|------|------|------|
| **管道** | 内核中的一块缓存。匿名管道用于亲缘进程，命名管道用于任意进程 | 单向（半双工），只能承载字节流，容量有限 |
| **消息队列** | 保存在内核中的消息链表 | 克服管道只能承载字节流的缺点，但读写需要用户态/内核态拷贝 |
| **共享内存** | 多个进程映射到同一块物理内存 | **最快的进程通信方式**，需要注意同步问题 |
| **信号量** | 表示资源数量，通过 P（-1）V（+1）操作控制访问 | 常用于进程同步 |
| **信号** | 操作系统向进程发送的软件中断通知 | 属于异步机制 |
| **Socket** | 套接字通信 | 可用于**不同机器**间的进程通信 |

## Q4. 线程通信方式

| 方式 | 说明 |
|------|------|
| **synchronized 同步** | 互斥锁保证同一时刻只有一个线程访问共享资源 |
| **共享内存 + volatile** | volatile 关键字保证变量的内存可见性 |
| **等待通知机制** | `wait()` / `notify()` / `notifyAll()`；`join()` 等待另一个线程结束 |
| **Lock + Condition** | `await()` / `signal()`，比 wait/notify 更灵活 |
| **管道通信** | `PipedInputStream` / `PipedOutputStream` |
| **BlockingQueue** | 阻塞队列实现生产者-消费者模式 |

## Q5. 死锁定义、条件和避免

**定义：** 两个及以上的线程持有自己的资源，又等待其他线程占用的资源，在无外力的情况下陷入无限期的相互等待状态。

**四个必要条件（缺一不可）：**
1. **互斥条件**：资源不能被多个线程同时使用
2. **请求并持有**：线程持有资源的同时请求其他资源
3. **不可剥夺条件**：线程已获得的资源不能被强制剥夺，只能自己释放
4. **循环等待条件**：多个线程形成一个循环等待链

**如何避免（破坏必要条件）：**
- 互斥条件：不可破坏（互斥是正常需求）
- 消除**请求并持有**：一个线程一次性请求需要的所有资源
- 消除**不可剥夺**：申请不到资源时，主动释放已持有的资源
- 消除**循环等待**：给资源排序，按固定顺序申请资源

**死锁预防算法：** 银行家算法（判断分配后是否处于安全状态）

## Q6. 内存页面置换算法

当物理内存不足，需要将某页换出时，选择哪一页的算法：

| 算法 | 说明 | 特点 |
|------|------|------|
| **OPT（最优置换）**| 置换未来最长时间不会被访问的页 | 理论最优，实际无法实现（无法预知未来）|
| **FIFO（先进先出）**| 置换最早进入内存的页 | 简单，但可能置换常用页（Belady 异常）|
| **LRU（最近最少使用）**| 置换最近最长时间没有被访问的页 | 性能好，常用，需要记录访问时间 |
| **Clock（时钟算法）**| 给每页设置访问位，循环扫描，访问位为 0 则置换 | LRU 的近似实现，开销小 |

## Q7. 虚拟内存

**定义：** 虚拟内存是一种内存管理技术，将**磁盘空间**作为内存扩展，逻辑上扩大了可用内存

**工作原理：**
- 程序运行时，只将**当前需要的部分**加载到物理内存
- 不需要的部分存储在磁盘（交换空间）
- 通过**页表**将虚拟地址映射到物理地址
- 访问未在内存中的页时，触发**缺页中断**，将需要的页从磁盘加载到内存

**段页式存储管理：**
- **段式**：按程序逻辑结构划分为段（代码段、数据段等），段内连续存储
- **页式**：将内存和程序都划分为固定大小的页，减少内存碎片
- **段页式**：先分段再分页，结合两者优点

**快表（TLB）：** 页表通常存储在内存中，每次地址转换都需要访问内存。TLB（Translation Lookaside Buffer）是 CPU 内的高速缓存，缓存最近使用的页表项，加速地址转换。
