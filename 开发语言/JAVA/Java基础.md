# Java 基础面试题

## Q1. Java 的特点和优点

**特点：**
- 面向对象（封装、继承、多态）
- 平台独立性（一次编译，到处运行，因为平台不同版本 JVM 解释 `.class` 字节码文件）

**优点：**
- 垃圾回收器，不需要开发人员进行内存管理
- 相比 C++，没有多继承和指针的复杂问题，完全面向对象
- 具有很多内置的类库，便于程序设计开发
- 提供防止恶意代码攻击的安全机制（数据边界检测、Bytecode 校验等）
- 支持网络编程（Web），如 Servlet、JSP 等
- 多线程
- 强大的生态社区

---

## Q2. Java 基本数据类型和引用类型

**八大基本数据类型：**
- 整型：`byte`(1字节)、`short`(2字节)、`int`(4字节)、`long`(8字节)
- 浮点型：`float`(4字节)、`double`(8字节)
- 字符型：`char`(2字节)
- 布尔类型：`boolean`

**引用类型：** 对一个对象的引用（数组、类、接口、String 等）

**存储区别：**
- 基本类型的变量值直接存储在**栈内存**中
- 引用类型的变量存储引用地址，真正的值存储在**堆内存**中

**装箱拆箱：**
- 拆箱：`i.xxxValue()` — 包装类型转基本数据类型
- 装箱：`XXX.valueOf()` — 基本类型转包装类

---

## Q3. 抽象类和接口的区别

**相同点：**
- 都不能被实例化，都在继承树的顶端，只能被其他类实现和继承
- 都可以包含抽象方法
- 抽象类定义：`abstract class`，只能被继承 `extends`
- 接口定义：`interface`，只能被实现 `implement`

**不同点：**
| 比较点 | 抽象类 | 接口 |
|--------|--------|------|
| 继承/实现 | 单继承 | 可实现多个接口 |
| 成员变量 | 静态变量、常量、成员变量 | 只能有常量 |
| 构造方法 | 可以有 | 没有 |
| 方法实现 | 可以实现普通方法 | 只能实现静态方法、默认方法 |

- 抽象类在业务编程上更像是一个模板，有自己的功能，也可以被优化补充
- 接口像是一种规范和要求，实现就要按照要求进行

---

## Q4. final 关键字

`final` 可以修饰**类、方法、变量**：
- `final` 修饰类：该类不能被继承
- `final` 修饰方法：该方法不能被重写，但子类可以使用该方法
- `final` 修饰变量：变量必须进行初始化，之后不能再进行赋值操作
  - `final` 修饰的引用变量，代表引用地址不可变，但对象内容是可以被改变的

---

## Q5. static 修饰符的用法

`static` 可以修饰**类、成员变量、初始化块、方法**，不可以修饰构造器：
- 由 `static` 修饰的成员称为**类成员**，可以通过类名访问，属于整个类
- 随着类加载时创建，比实例成员更早存在
- 类成员可以访问实例成员，但实例成员不能访问类成员
- 静态方法中没有 `this` 关键字，`this` 关键字是随着对象创建而存在的

---

## Q6. String 类、new String 和字符串直接量

- `String` 类是被 `final` 修饰，不可以被继承，且重写了 `equals()` 方法
- **字面量创建**：直接创建字符串，存储在**常量池**中
- **new 创建**：先查找常量池中是否存在字符串，不存在则常量池创建一个，并 `new` 一个 String 对象存储在**堆内存**中，占用更多内存

**常用方法：**
- 大小写转换：`toUpperCase()`、`toLowerCase()`
- 获取下标字符：`charAt(index)`
- 获取子字符串：`subString(begin, end)`
- 去除首位空格：`trim()`
- 首次出现下标索引：`indexOf(index)`
- 替换内容：`replaceFirst(regex, replacement)`、`replaceAll(regex, replacement)`

## Q7. String、StringBuffer、StringBuilder 的区别

| 特性 | String | StringBuffer | StringBuilder |
|------|--------|-------------|---------------|
| 可变性 | 不可变 | 可变 | 可变 |
| 线程安全 | 安全（不可变）| 安全（synchronized）| 不安全 |
| 性能 | 低 | 中 | 高 |
| 适用场景 | 少量操作 | 多线程 | 单线程 |

- 三者都被 `final` 修饰，不可被继承
- 可变字符串类可以通过 `append()`、`insert()`、`reverse()` 改变字符串
- 可变字符串可通过 `toString()` 转换为 String 对象

## Q8. == 和 equals() 的区别

- `==`：对于**基本数据类型**比较值是否相等；对于**引用类型**比较引用地址是否相等
- `equals()`：`Object` 类提供的方法，没有重写时默认使用 `==` 进行比较，可根据需求重写 `equals()` 比较对象内容

---

## Q9. hashCode() 和 equals() 的关系

- `hashCode()` 通过哈希算法获取哈希值；`equals()` 比较对象是否相等
- **默认约定：**
  - `hashCode()` 相等的两个对象，`equals()` 判断结果**不一定**相等
  - `equals()` 相等的两个对象，`hashCode()` 哈希码**一定**相等
- 重写 `equals()` 方法，也要重写 `hashCode()` 方法，避免发生 `equals()` 相等但 `hashCode()` 不相等的矛盾

## Q11. 包装类的自动拆箱和自动装箱

- **自动装箱**：基本数据类型直接赋值给包装类对象
- **自动拆箱**：包装类对象可以直接赋值给基本数据类型
- Java 是纯粹的面向对象语言，当需要用到 Object 类型参数时，通过自动装箱对基本数据类型进行方便的转换
- **注意：** 不同的包装类不可以直接进行比较；可以将 Integer 和 Double 都转换为 double 进行比较

## Q12. Java 异常处理机制

**三个主要方面：** 处理异常、抛出异常、异常跟踪栈

**处理异常关键字：**
- `try`：业务逻辑代码
- `catch`：捕获异常并处理
- `finally`：资源回收

**主动抛出异常：**
- `throws`：可抛出多个异常类，只能在方法签名中使用，不一定发生
- `throw`：在方法体中条件判断抛出异常实例，一定抛出且只能抛出一个异常

**流程：** 主动抛出的异常交给上级调用者处理。如果一直没有得到处理，到达 JVM 层面，JVM 打印异常跟踪栈信息并停止程序运行

## Q13. 面向对象三大特征

**封装：** 将对象的状态信息隐藏在对象内部，外部程序只能通过类提供的方法进行操作和访问，规范了使用者行为，提高代码的可维护性

**继承：** 提高了代码的复用性，提升代码编写的效率。也增加了耦合性，修改父类代码对子类影响较大

**多态：** 在继承的基础上，要求子类重写了父类的方法。主要形式是**父类的引用指向了子类的对象**

---

## Q14. 重载和重写的区别

| 特性 | 重载（Overload）| 重写（Override）|
|------|----------------|----------------|
| 范围 | 同一个类中 | 父子类之间 |
| 方法名 | 相同 | 相同 |
| 参数 | 不同 | 相同 |
| 返回值 | 可以不同 | 子类返回值范围 ≤ 父类 |
| 异常 | 可以不同 | 子类抛出异常范围 ≤ 父类 |
| 访问修饰符 | 可以不同 | 子类访问修饰符 ≥ 父类 |

- `final`、`private` 修饰的方法不可以进行重写
- 构造方法也不可以重写

## Q15. 访问修饰符

权限范围：`private` < `default` < `protected` < `public`

| 修饰符 | 本类 | 同包 | 子类 | 其他包 |
|--------|------|------|------|--------|
| `private` | ✓ | ✗ | ✗ | ✗ |
| `default` | ✓ | ✓ | ✗ | ✗ |
| `protected` | ✓ | ✓ | ✓ | ✗ |
| `public` | ✓ | ✓ | ✓ | ✓ |

- `private`、`default`、`public` 都可以修饰类、方法、变量
- `protected` 不可用于修饰类

## Q16. 泛型与泛型擦除

**泛型的作用：** 引入泛型后，规定了集合可以接收的数据类型，编译器会对数据类型进行安全检查，避免运行时类型转换错误

**泛型擦除：** Java 泛型是**伪泛型**，编写的泛型在**编译阶段会被擦除**，然后进行相应的类型转换

## Q17. 反射

- 反射机制可以在程序**运行时动态**地获取类的 `.class` 对象，对类信息操作
- 可以在运行时动态地创建类的对象实例，获取属性和方法等操作
- 可以生成**动态代理类**和动态代理对象

**优缺点：**
- 优点：提高了代码的灵活性
- 缺点：破坏了 Java 的封装性，有一定的安全隐患，且性能低

---

## Q18. 面向对象与面向过程的区别

| 对比 | 面向过程 | 面向对象 |
|------|---------|---------|
| 思维方式 | 分析解决问题的步骤，用函数逐步实现并依次调用 | 将问题拆分为各个对象，描述对象在事务中的行为 |
| 代码复用 | 较低 | 高，通过继承、封装、多态实现 |
| 扩展性 | 低 | 高，易于扩展和维护 |
| 适用场景 | 流程固定的小程序 | 复杂系统，需要扩展和复用 |

**面向对象三大特性：**
- **封装**：将对象属性私有化，并提供供外界访问的方法
- **继承**：在已存在的类基础上创建新的类，增加新的属性和方法，实现代码复用
- **多态**：允许不同对象对同一方法表现出不同行为（通过继承和方法重写实现，使程序更灵活，降低耦合度）

## Q19. 基本数据类型存储在栈还是堆？

- **声明在方法中**（局部变量）：存储在**栈帧**中
- **声明为类的成员变量或 static 变量**：存储在**堆**中

> 注意：这与"基本类型存在栈"的简化说法不同，存储位置取决于变量的声明位置，而非类型本身。

## Q20. String 为什么是不可变的？

**三个原因：**

1. **安全性**：String 多用来做 URL、文件路径、用户密码、Socket 中的主机端口号，保证不可变更安全
2. **字符串池的实现依赖不可变性**：如果 String 可变，常量池中某一引用修改了值，其他引用也会受到影响，字符串池就无法工作
3. **Hash 缓存**：字符串不可变使 Hash 码具有唯一性，创建时就被缓存。String 适合做 Map 的键，处理速度快于其他键对象

## Q21. Integer 缓存机制

- `Integer.cache` 存储的范围是 **-128 ~ 127**
- `Integer a = 127` 自动装箱调用 `Integer.valueOf()`，会用到缓存池
- **范围内**：从缓存池取数，`==` 时返回 `true`
- **范围外**：`new` 一个新对象，`==` 时返回 `false`
- `Integer` 与 `int` 比较时会**自动拆箱**为 `int`，比较的是值

```java
Integer a = 127, b = 127;
a == b; // true（缓存池）

Integer c = 128, d = 128;
c == d; // false（新对象）

Integer e = 200;
e == 200; // true（自动拆箱比较值）
```

## Q22. 哈希冲突的解决办法

1. **开放定址法**：冲突时在哈希表中寻找下一个空位
2. **链地址法**：在冲突槽位使用链表存储所有冲突元素（HashMap 采用此方式）
3. **再哈希法**：使用另一个哈希函数重新计算哈希值
4. **建立公共溢出区域**：将冲突的元素放入独立的溢出区域

## Q23. JVM、JDK、JRE 的区别

| 名称 | 全称 | 作用 |
|------|------|------|
| JVM | Java Virtual Machine | Java 虚拟机，运行 .class 字节码文件，实现跨平台 |
| JRE | Java Runtime Environment | Java 运行环境 = JVM + 核心类库，运行 Java 程序所需 |
| JDK | Java Development Kit | Java 开发工具包 = JRE + 编译器 javac + 调试工具等，开发 Java 程序所需 |

**包含关系：** JDK ⊃ JRE ⊃ JVM
