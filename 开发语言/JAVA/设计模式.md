# 设计模式面试题

## Q1. 常见设计模式概览

共 23 种设计模式，常见有：单例模式、工厂模式、代理模式、适配器模式、模板方法模式

**在 Spring 中的体现：**
| 设计模式 | Spring 中的体现 |
|---------|---------------|
| 单例模式 | Spring 中的 Bean 默认就是单例模式 |
| 工厂模式 | Spring 中的 BeanFactory |
| 代理模式 | AOP（面向切面编程）底层使用动态代理实现 |
| 适配器模式 | SpringMVC 根据处理器规则适配处理器 |
| 模板方法模式 | MyBatis 中的 JdbcTemplate 使用了模板方法模式 |

## Q2. 单例模式

**定义：** 类中只有一个实例，避免了多次创建和销毁的开销

**缺点：** 没有子类，不能扩展

**分类：**

### 饿汉式单例模式
在**类加载时**就创建对象实例并初始化

```java
public class Single {
    private static Single instance = new Single();
    
    private Single() {}
    
    public static Single getInstance() {
        return instance;
    }
}
```

### 懒汉式单例模式（线程不安全版）
在类被引用时才创建实例

```java
public class Single {
    private static Single instance = null;
    
    private Single() {}
    
    public static Single getInstance() {
        if (instance == null) instance = new Single();
        return instance;
    }
}
```

### 懒汉式单例模式（线程安全版）
在 `getInstance()` 方法上加 `synchronized`

```java
public class Single {
    private static Single instance = null;
    
    private Single() {}
    
    public static synchronized Single getInstance() {
        if (instance == null) instance = new Single();
        return instance;
    }
}
```

### 双重锁定（推荐）

```java
public class Single {
    private static volatile Single instance = null;
    
    private Single() {}
    
    public static Single getInstance() {
        if (instance == null) {
            synchronized (Single.class) {
                if (instance == null) {
                    instance = new Single();
                }
            }
        }
        return instance;
    }
}
```

### 静态内部类（推荐）

```java
public class Single {
    private Single() {}
    
    private static class SingleHolder {
        private static final Single INSTANCE = new Single();
    }
    
    public static Single getInstance() {
        return SingleHolder.INSTANCE;
    }
}
```

## Q3. 工厂模式

**定义：** 将创建对象实例的细节封装在一个函数方法中

**三种工厂模式：**

### 简单工厂
工厂类中根据传入的参数返回不同的实例（不符合开闭原则，反射机制可以解决）

```java
public class Computer {}
public class AdolComputer extends Computer {}
public class DellComputer extends Computer {}

public class FactoryComputer {
    public enum ComputerType { ADOL, DELL; }
    
    static Computer create(ComputerType computerType) {
        if (computerType.equals(ComputerType.ADOL))
            return new AdolComputer();
        if (computerType.equals(ComputerType.DELL))
            return new DellComputer();
        else return null;
    }
}
```

### 工厂方法
通过实现工厂接口中的方法创建不同的实例对象，但只能创建一维的对象

```java
public interface FactoryComputer {
    Computer createComputer();
}

public class AdolFactoryComputer implements FactoryComputer {
    public Computer createComputer() {
        return new AdolComputer();
    }
}

public class DellFactoryComputer implements FactoryComputer {
    public Computer createComputer() {
        return new DellComputer();
    }
}
```

### 抽象工厂
工厂方法的延伸，可以创建多维的对象

```java
public interface Factory {
    Computer createComputer();
    Server createServer();
}

public class AdolFactory implements Factory {
    public Computer createComputer() { return new AdolComputer(); }
    public Server createServer() { return new AdolServer(); }
}

public class DellFactory implements Factory {
    public Computer createComputer() { return new DellComputer(); }
    public Server createServer() { return new DellServer(); }
}
```

## Q4. 代理模式

**定义：** 核心业务与辅助业务的细分（好比明星和经纪人的工作）

**在 Spring AOP 中：** 底层使用动态代理，对多个类中共性需求的代码单独封装，再配置织入切面位置

---

## Q5. 适配器模式

**定义：** 将一个类的接口转换为客户希望的另一个接口

**在 SpringMVC 中：** 根据处理器的规则适配处理器

---

## Q6. 模板方法模式

**定义：** 解决代码重复问题，定义算法骨架，将某些步骤延迟到子类中实现

**在 MyBatis 中：** JdbcTemplate 使用了模板方法模式
