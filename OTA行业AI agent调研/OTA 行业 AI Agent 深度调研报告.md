

## 第一部分：未来 OTA 与桌面 Agent 的 4 个终极形态

### 1. 交互升级：从“搜 App”变成“生成可视化大屏 (Generative UI)”
*   **解决痛点：** 以前要在不同 App 切换，查完机酒还要自己写文档。纯对话（LUI）看地图和时刻表很累。
*   **未来形态：** 桌面 Agent 在屏幕侧边直接生成一个**“类似 Notion 的可视化行程大屏”**。
*   **交互逻辑：** 用户说话控制意图（LUI），大屏负责展示信息（GUI）。想改哪里点哪里，局部刷新，自动重算预算。

### 2. 决策重构：从“单人定夺”变成“群聊协作 (Multiplayer AI)”
*   **解决痛点：** 旅游通常是多人的，但目前的 OTA 只支持一个人订，很难统一大家的意见。
*   **未来形态：** Agent 潜伏在微信/飞书群里，自动提取大家的要求（“我不吃辣”、“我要直飞”），综合计算出大家都能满意的方案。

### 3. 价格博弈：从“公开比价”变成“私有会员价”
*   **解决痛点：** 系统底层 Agent 会全网比价，OTA 如果只给公开价，就会沦为廉价的数据来源，失去控制权。
*   **未来形态：** OTA 必须提供**私有 API**。Agent 会引导用户登录会员，调用只有该用户能享受的“白金会员价”或“私有优惠券”，在比价中胜出。

### 4. 销售创新：从“硬弹窗”变成“自然语境推荐”
*   **解决痛点：** 以前在支付页强推租车/保险，用户很反感。
*   **未来形态：** Agent 全程陪伴。比如发现你要去日本，它会自然提醒：“办签证要 5 天，你时间刚好够，要现在一键下单吗？”

## 第二部分：OTA 平台超越飞猪的 5 大核心战役

### 核心战役一：抢占标准 —— 研发并开源“生成式行程组件库”
*   **目的：** 让所有 AI 渲染出来的行程界面，都使用我们定义的精美样式。
*   **动作 1：** 制定 `Generative UI` 标准：提供机票、酒店、地图等前端组件包。
*   **动作 2：** 支持局部修改：用户说“不喜欢这间房”，组件能直接响应修改，不用大模型重写整篇攻略。

### 核心战役二：打通闭环 —— 开放“带身份鉴权”的下单 API
*   **目的：** 不只是能查，还要能一句话直接买。
*   **动作 1：** 私有身份注入：用户授权后，返回的是基于会员等级的绝对底价。
*   **动作 2：** 读写闭环：开放创单和支付接口，支持指纹/免密支付，实现“一句话下单”。

### 核心战役三：社交协同 —— 研发“多人意图聚合引擎”
*   **目的：** 搞定群聊里的复杂决策，解决“众口难调”的问题。
*   **动作 1：** 开发群组会话 API：支持向一个 ID 注入多个人的偏好，运行“冲突消解算法”。
*   **动作 2：** 协作式画布：发一个卡片到群里，大家都能在上面点点改改，实时看到总价变动。
*   **动作 3：** 一键分账（AA 支付）：行程定好后，Agent 生成账单，每个人点开付自己的钱，付齐了自动出票。

### 核心战役四：商业增值 —— 构建“行中履约引擎”
*   **目的：** 提升高毛利（签证、接送机）销量，管好行中突发状况。
*   **动作 1：** 前置交叉销售：基于推理，精准下发签证、快速通关等高利润服务。
*   **动作 2：** 行中管家：如果航班延误或景点关门，Agent 主动推退改方案或周边替代方案。

### 核心战役五：流量分发 —— 开辟 TaaS (旅行即服务) 创作者网络
*   **目的：** 去中心化，让全网都能帮我们卖货。
*   **动作 1：** 赋能博主（C 端）：为小红书博主提供 AI 套件，生成的攻略自带预订代码，用户看攻略时找 Agent 就能下单，博主拿佣金。
*   **动作 2：** 渗透企业（B 端）：接入钉钉/飞书底层，员工一句话提报，AI 自动对齐公司报销标准并订好合规机酒。
